## Installation

Install RunAgent CLI using pip:

```bash
pip install runagent
```

Verify the installation:

```bash
runagent --help
```

## Create Your First Agent

Let's create a simple problem-solving agent using LangGraph:

<Steps>
  <Step title="Initialize the Project">
    ```bash
    runagent init my-first-agent --framework langgraph
    cd my-first-agent
    ```
    
    This creates a new directory with:
    - Pre-configured agent template
    - `runagent.config.json` configuration file
    - Required dependencies in `requirements.txt`
    - Example agent implementation
  </Step>
  
  <Step title="Install Dependencies">
    ```bash
    pip install -r requirements.txt
    ```
  </Step>
  
  <Step title="Configure Environment">
    Create a `.env` file with your API keys:
    
    ```bash
    echo "OPENAI_API_KEY=your-api-key-here" > .env
    ```
    
    <Warning>
      Never commit `.env` files to version control. Add it to `.gitignore`.
    </Warning>
  </Step>
  
  <Step title="Test Locally">
    Start the local development server:
    
    ```bash
    runagent serve .
    ```
    
    Your agent is now running at `http://localhost:8000`
  </Step>
</Steps>

## Test Your Agent

<Tabs>
  <Tab title="Using cURL">
    ```bash
    # Test the health endpoint
    curl http://localhost:8000/health

    curl -X POST http://localhost:8000/invoke \
      -H "Content-Type: application/json" \
      -d '{

        "num_solutions": 3
      }'
    ```
  </Tab>
  
  <Tab title="Using Python">
    ```python
    import requests
    
    # Test invocation
    response = requests.post(
        "http://localhost:8000/invoke",
        json={
            "query": "How do I fix a leaking faucet?",
            "num_solutions": 3
        }
    )
    
    print(response.json())
    ```
  </Tab>
  
  <Tab title="Streaming Response">
    ```python
    import requests
    
    # Stream responses
    response = requests.post(
        "http://localhost:8000/stream",
        json={"query": "Explain photosynthesis"},
        stream=True
    )
    
    for line in response.iter_lines():
        if line:
            print(line.decode('utf-8'))
    ```
  </Tab>
</Tabs>

## Understanding the Project Structure

Your agent project contains:

```
my-first-agent/
├── agents.py              # Your agent implementation
├── runagent.config.json   # RunAgent configuration
├── requirements.txt       # Python dependencies
├── .env                   # Environment variables (create this)
└── README.md             # Project documentation
```

### The Configuration File

The `runagent.config.json` file defines how RunAgent interacts with your agent:

```json
{
  "agent_name": "my-first-agent",
  "description": "A simple problem-solving agent",
  "framework": "langgraph",
  "version": "1.0.0",
  "agent_architecture": {
    "entrypoints": [
      {
        "file": "agents.py",
        "module": "app.invoke",
        "type": "generic"
      },
      {
        "file": "agents.py",
        "module": "app.stream",
        "type": "generic_stream"
      }
    ]
  },
  "env_vars": {
    "OPENAI_API_KEY": "${OPENAI_API_KEY}"
  }
}
```

## Deploy Your Agent (Local)

For local deployment with persistence:

```bash
runagent deploy . --local
```

This will:
- Validate your configuration
- Set up the local environment
- Start your agent with auto-restart capability
- Provide you with a deployment ID

## Use the Python SDK

Once deployed, interact with your agent using the Python SDK:

```python
from runagent import RunAgentClient

# Initialize the client
client = RunAgentClient(agent_id="your-deployment-id")

# Simple invocation
result = client.run_generic({
    "query": "What's the weather like today?",
    "location": "San Francisco"
})
print(result)

# Streaming response
for chunk in client.run_generic_stream({
    "query": "Write a short story about AI"
}):
    print(chunk, end="", flush=True)
```

## Common Issues

<AccordionGroup>
  <Accordion title="Port 8000 is already in use">
    Specify a different port:
    ```bash
    runagent serve . --port 8080
    ```
  </Accordion>
  
  <Accordion title="Module not found errors">
    Ensure you're in the correct directory and have installed dependencies:
    ```bash
    pwd  # Should show your agent directory
    pip install -r requirements.txt
    ```
  </Accordion>
  
  <Accordion title="API key errors">
    Check that your `.env` file exists and contains valid API keys:
    ```bash
    cat .env  # Should show your API keys
    ```
  </Accordion>
</AccordionGroup>
